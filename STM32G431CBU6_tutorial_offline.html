4) Pines y funciones alternativas (AF)

Un pin puede ser GPIO, funcion alternativa o analogico. La disponibilidad exacta depende del encapsulado y de la tabla AF del datasheet.

Regla practica

  • No todos los pines pueden hacer todo.
  • SWD (debug) se reserva al principio.
  • SPI pantalla y ADC suelen competir por pines en placas compactas.

Conflictos comunes

  • Elegir pin de ADC que luego quieres usar como salida digital.
  • Perder el pin de UART debug por reasignacion tardia.
  • Olvidar ajuste de velocidad GPIO para SPI rapido.

Flujo recomendado: CubeMX primero para mapa inicial; ajuste manual despues con criterio de debug y ruido.

5) Reloj del sistema y PLL

Todo parte del clock tree. Si no lo entiendes, no controlas timer, ADC ni comunicaciones.

Camino tipico

Fuente (HSI/HSE) -> PLLM -> PLLN -> PLLR -> SYSCLK

Formulas base de PLL (familia STM32)

fVCO_in = fPLL_source / M
fVCO_out = fVCO_in * N
fPLL_R = fVCO_out / R

Donde M, N, R se configuran en registros PLL. En G4 es comun usar PLL para llegar a 170 MHz.

Relacion con timers

El reloj del timer no siempre es igual al PCLK. Segun prescaler APB, los timers pueden recibir factor extra (x2 en ciertas configuraciones). Verificalo siempre en Clock Configuration de CubeMX y en RM.

Consejo: fija primero un "reloj de trabajo" simple (por ejemplo timer counter a 1 MHz) y disena todo desde ahi.

6) Timers: tipos, bits, registros y vinculo al reloj

Tipos de timer (familia G4, disponibilidad segun encapsulado)

Tipo Ejemplos comunes Bits Uso recomendado
Basico TIM6, TIM7 16-bit Base temporal, disparo ADC, tareas periodicas.
General TIM2, TIM3, TIM4 TIM2 suele 32-bit; otros 16-bit PWM, captura, medicion de periodo/frecuencia.
Avanzado TIM1 (y segun modelo TIM8) 16-bit PWM avanzado, dead-time, break, control de potencia/motor.
Control/complementarios TIM15/TIM16/TIM17 16-bit Canales extra para control fino o sincronias auxiliares.

Grafico de registros del timer (vista mental)

Reloj TIMxCLK
   |
   v
+-------+     +-------+     +--------+
| PSC   | --> | CNT   | --> | ARR    | --> Evento Update (UIF)
|Divide |     |Cuenta |     |Tope    |
+-------+     +-------+     +--------+
      \                         |
       \                        v
        \------------------> DIER.UIE -> IRQ

Otros registros clave:
- CR1: habilitacion, direccion, autoreload preload
- EGR: generar eventos por software (UG)
- SR: flags (UIF, CCxIF...)
- CCMR/CCER/CCR1..CCR4: compare/capture/PWM
- BDTR (timers avanzados): dead-time, break, main output enable

Formula de frecuencia

f_counter = f_TIMxCLK / (PSC + 1)
f_update = f_counter / (ARR + 1)
T_update = 1 / f_update

Ejemplo numerico para 5 kHz

Si f_TIMxCLK = 170 MHz, puedes usar PSC = 169 para obtener contador a 1 MHz, y luego ARR = 199 para update a 5 kHz.

7) Interrupciones y prioridades

El NVIC permite priorizar eventos. Una ISR larga destruye determinismo, especialmente con captura de impacto.

Interrupcion Uso en proyecto Regla de ISR
EXTI Boton reset/nueva sesion Debounce + bandera, nada de logica pesada.
TIMx update Tick fijo o sincronias Marcar evento; procesar fuera.
ADC DMA half/full Captura de buffer Confirmar tramo listo y salir.
SPI DMA Pantalla o ADC externo SPI No bloquear otras ISR de adquisicion.
Orden orientativo de prioridad: adquisicion (ADC/DMA) > timer control > comunicaciones > UI.

8) ADC y DMA en adquisicion real

Para forma de onda confiable, el disparo del ADC debe venir de timer, no de software en bucle.

TIM6/TRGO @ fs fija ---> ADC1 conversion ---> DMA ---> Buffer RAM [N muestras]
                                             |
                                             +--> IRQ fin de buffer (flag done)

Cadena de conversion

Counts ADC -> Voltaje (Vref) -> Fuerza (calibracion)
F = gain * (V - offset)

Con carga de impacto, el offset (tara) y la ganancia calibrada son obligatorios para resultados utilizables.

9) Como pensar los registros sin perderte

Metodo mental

  • RCC: habilita reloj del periferico.
  • Config: define modo (CRx/CFGR/CCMR...).
  • EGR: fuerza update si hace falta.
  • DIER/NVIC: habilita interrupcion.
  • SR: lee/limpia flags correctamente.

Patron de fallo tipico

  • No habilitar clock RCC del periferico.
  • Configurar registros con timer corriendo.
  • No limpiar UIF/CCxIF y repetir ISR.
  • Asumir valores por defecto sin verificar.
// Ejemplo minimo de timer por registros (conceptual)
RCC->APB1ENR1 |= RCC_APB1ENR1_TIM6EN;   // 1) Clock periferico
TIM6->PSC = 169;                        // 2) Prescaler
TIM6->ARR = 199;                        // 3) Periodo
TIM6->EGR = TIM_EGR_UG;                 // 4) Forzar update de registros sombra
TIM6->DIER |= TIM_DIER_UIE;             // 5) Interrupcion update
TIM6->CR1 |= TIM_CR1_CEN;               // 6) Arrancar contador

10) Formulas utiles para calculo rapido

N = fs * ventana_s
dt = 1 / fs
Impulso J ~= Sum(F[i] * dt)
RFD[i] ~= (F[i] - F[i-1]) / dt
Objetivo Dato Resultado
80 ms a 5 kSPS 0.08 s * 5000 400 muestras
60 lecturas en 50 ms 60 / 0.05 s 1200 SPS
Timer 170 MHz -> 5 kHz PSC=169, ARR=199 1 MHz / 200 = 5 kHz

ISR EXTI con debounce

void HAL_GPIO_EXTI_Callback(uint16_t GPIO_Pin)
{
    static uint32_t last_ms = 0;
    uint32_t now = HAL_GetTick();

    if (GPIO_Pin == GPIO_PIN_13 && (now - last_ms) > 50U) {
        g_reset_requested = 1;  // Solo bandera
        last_ms = now;
    }
}

12) Tips and tricks para avanzar rapido

  • Reserva siempre SWD y un UART de debug.
  • Valida timing con osciloscopio (pin toggle) antes de fiarte de logs.
  • No dibujes pantalla dentro de ISR.
  • Define unidades internas: N, ms, N*s. Evita mezclar con kgf en calculo.
  • Versiona parametros de calibracion junto al firmware.
  • Si hay clipping, el pico no es valido aunque el numero "sea grande".
